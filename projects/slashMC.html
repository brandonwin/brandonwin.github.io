slashMC

Managing your server with friends is that easy.

slashMC is a desktop and mobile application that makes operating a Minecraft server easy. slashMC is a graphical user interface (GUI) that strikes a balance between visual appeal and powerful functionality, easing new and experienced administrators into operating their own servers.

Product

Desktop App (GUI), Mobile App

Technologies

Adobe XD, Adobe Photoshop, Adobe Illustrator

Key Services

UX Design, UX Research, UI Design, Wireframing

The Problem

Minecraft server administrators are restricted almost exclusively to operating their servers through the command prompt. Other GUIs are either considered janky or too resource-draining.

The Solution

Design an alternative GUI for server operation, striking a medium between traditional command prompt based operation and modern user interface design.

The User

Aaron, 18, High schooler

Aaron has played Minecraft with his friend group for several years, and he's typically designated as the server owner. He loves playing with his friends on their own server, but he's had to exclusively use the command prompt since there aren't any functionally powerful or visually attractive alternatives.

"I've been playing Minecraft for years with friends, but there are never good alternatives to the command prompt. I don't want to have to interact with an interface this ancient in 2022."

Motivations

  • Have the same amount of control and power as a command prompt
  • Interact with the GUI through a way other than typing
  • Monitor his server through an alternative means as a convenience

Discoveries

Early Findings

  • Utilizing a GUI other than the command prompt has to be worth it. Choosing to use a GUI is a huge pros-versus-cons choice for users, and the product has to have enough of a significant advantage/pull to invest in it.
  • Experienced administrators will still want access to more traditional forms like the command prompt, while newer users are interested in CTAs and button interactions.
  • Nearly all information should be readily accessible on the main screen.

Product Pivots

  • Determine what sets slashMC apart.
  • Create a companion app for slashMC to differentiate itself from its competitors and satisfy a convenience for users.
  • Develop design methods to tell server operation information through color.

Iteration

Wireframing

I decided on a segmented interface with different panels dedicated to separate functions in order to keep the interface organized for newcomers and veterans. Everything that an operator would want is immediately availablet with little to no fluff (a key requirement that many server operators voiced that they would want).

Style Guide

I designed the background colors to be dark since I didn't want the interface to be too bright or harsh on the eyes, since slashMC could be running 24/7 for many users. I also wanted the text and interactions to have very simple color associations (green to start processes, red to stop, etc.). Stylistically, icons and fonts are simple in order to reflect a more modern, Apple-esque interface.

Learnings

  1. Blind user testing is a great way to test the "sensibility" of your app. I was suggested to do blind online user testing, which allowed me to test whether the average user could understand what the buttons on my interface actually do (which would help me design for experienced users as well).
  1. "Design is built from constraints." I realized that designing mobile first is much better than designing web first. The restricted amount of space on the actual screen makes you make tougher and more restrictive decisions on your designs, but I believe that it ultimately creates a better product. Web to mobile is rougher than mobile to web, and I discovered my preference for designing mobile to web.
