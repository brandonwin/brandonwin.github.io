Managing your server should be easy, but it isn't.

Most Minecraft server administrators use the command prompt to operate their servers. Why not a more sophisticated interface? Because there simply isn't a better alternative out there. Other GUIs are either difficult to adjust to, resource-intensive, or just not worth the switch.

Problem

Minecraft server administrators don't have a user-friendly and resource-friendly interface for managing their servers.

Solution

Design a GUI for server operation that strikes the balance between traditional operation and modern design.

Dashboard design

When I was designing the interface, I decided on a dashboard interface with panels for different features. I felt that this would keep the interface organized and easy to read for newcomers and veterans alike.

Style guide

I chose for the color palette to be a mix of blacks an greys since I didn't want the interface to be bright or harsh on the eyes. I wanted the text colors and interactions to have common UX associations (green to start, red to stop, etc.).

Research

Using a GUI instead of a command prompt has to be worth it for users.
Experienced users will want access to traditional forms of operation, while newer users will want buttons and CTAs.
Users want important information to be available on the main screen.

Direction

Design to set slashMC apart with unique features that aren't traditionally available.
Use design styles to communicate through color and font.
Companion app?

User Persona

Aaron, 18

Aaron has played Minecraft with his friend group for several years, and he's typically designated as the server owner. He loves playing with his friends on their own server, but he's had to exclusively use the command prompt since there aren't any functionally powerful or visually attractive alternatives.

Motivations

To have a better visual experience that doesn't sacrifice on functionality.
To have an easier time managing and monitoring his server.
"I've been playing Minecraft for years with friends, but there are never good alternatives to the command prompt. I don't want to have to interact with an interface this ancient in 2022."

Learnings

  1. Blind user testing is a great way to test the sensibility of your designs. I conducted this testing online with random users in my target audience, and I was able to test whether they could understand my iconography and layout.
  2. Design is built from constraints. I learned that designing mobile first (when available) helped my design process much more by constraining the amount of space I had in my frames, inevitably making a better product.
